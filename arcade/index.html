SJSU Arcade Launcher

A light-weight launcher for the SJSU arcade cabinets. Used to display, start, and monitor the games put on the arcade cabinets.

The Game Dev Club at San Jose State University created a couple arcade cabinets. The arcade cabinets are pretty much wooden shells that contain computers and monitors inside them (like most arcade cabinets). Joysticks are connected to the computer through USB. The computer then launches the video games.

The coolest thing about this arcade cabinet is that all the games on it are student-made.

The Launcher

The launcher is what glues all the video games on the arcade cabinet together. It is the screen you see when you scroll through the games. It is the what you use to start a game up.

Monitoring Games

The launcher monitors if a game executable has successfuly launched or if it has just closed. When it detects that a game has launched then it minimizes itself to allow the game to properly render. When the game has closed, the launcher opens back up.

Games that aren't designed for the arcade cabinets or are using legacy controls have Joy2Key bindings that remap the joystick keys. The launcher will activate those Joy2Key bindings based on what game has just launched.

The launcher has two methods of monitoring processes.

The first method of monitoring involves polling. It makes a request to the operating system for a list of all running executables, and matches it with the game executable name. It is slow but is the most stable way of monitoring a game.

The second method doesn't actually monitor the game at all. It launches the game as a subprocess. This is the fastest way of detecting when the process has started and when it has ended since it gets a message direct from the operating system. However, it doesn't work for certain games. Games using Game Maker create a helper executable that unzips its assets and launches a new process which has the actual game in it. The monitor can only monitor the helper executable, which only opens for a second.

Collecting Data

The launcher collects play-count and play-time data and outputs it into a JSON file. You are then able to visualize the data on a pi chart.

The Technology

The CSS and animations are done using LESS

The launcher application itself uses node-webkit / NW.js. NW.js allows developers to create standalone applications using web technologies. NW.js is a gutted web-browser. I call it a "gutted" web-browser because it's essentially still a web-browser but without extensions, address bars, and security.

NW.js allows me to create a webpage for the display and use node.js modules to launch and manage processes. NW.js also allows me to read and write to any files I want, which I use to save data collected.
